스마트폰 모바일웹을 이용하여 아래와 같은 조건의 기능을 수행하는 웹어플리케이션을 만들고자 한다.

[개발환경]
frontend : bootstrap, css를 이용하는 html기반 모바일 웹어플리케이션
backend : nodejs , sqllite 이용하여 사용자 300명이내의 소규모 관리기능을 포함하는 was 

[인증 기능 요구사항]
1. 사용자는 홈페이지의 특정 path 이하 페이지에 접근하는 경우 로컬스토리지에 jwt 형식으로 저장된 인증key 존재 여부에 따라 인증여부를 확인한다.
특정path : /app/event/*
스토리지 인증키 : forumUser

1-1. 해당 키가 없으면 인증절차로 이동하고 사용자id, 4자리 휴대전화 뒷번호를 입력받아 인증절차를 거친다.
 인증절차 : backend에서 아래 api 호출 후 호출결과에 따라 db에 인증정보 저장하고 frontend에서는 forumUser 키로 로컬스토리지에 저장한다.
1-2. 인증정보는 JWT 형식으로 발급되며, 토큰 만료시간은 3개월이다.
 - 토큰내용은 userid 등의 정보가 포함되어선 안된다.
 - backend에 저장된 특정 암호키만 저장하고 해당 암호키를 기반으로 복호화 하여 사용하여야 한다
1-3. 로그아웃을 하는 경우 해당 토큰을 삭제하여 보안성을 확보하고 홈페이지 header에 표시되는 로그인 사용자의 정보를 제거한다.
1-4. 인증이 완료된 사용자는 token을 이용하여 backend api를 호출하여 데이터를 조회하여 인증된 사용자의 사용자명, 부서명, 직급을 홈페이지 header에 표시한다.
1-5. 인증절차 및 api 호출시 네트워크 오류나 처리 오류가 발생하는 경우 사용자에게 적절한 오류 메세지를 표시한다.
1-6. 인증절차 및 api 호출 시 네트워크 오류가 발생할 경우, 사용자에게 적절한 오류 메시지를 표시한다.

[Authentication]

[업무 요구사항]
1. 1일간 실시되는 KB금융 테크포럼에 대한 랜딩페이지이다
2. 해당 페이지는 행사 안내, 일정, 연사소개, 부스소개, 참가신청, event 등의 섹션으로 구성된다
3. 행사 안내 섹션에는 행사 개요, 일시, 장소 등의 정보가 포함된다
4. 일정 섹션에는 시간대별 세션 정보가 포함되며 행사 중계의 youtube 영상 링크가 첨부된다.
5. 연사소개 섹션에는 연사의 사진, 이름, 소속, 소개글이 포함된다
6. 부스소개에서는 해당 부스에서 제공하는 서비스 및 체험행사에 대한 설명이 포함된다
7. 참가신청 섹션에는 사전접수 폼이 포함되어 상위 설명된 인증 절차를 진행한다
8. event 섹션에는 경품 이벤트 안내 및 참여 방법이 포함되며 인증 필수요구 페이지이다.
9. 각 섹션은 모바일웹 환경에 최적화되어야 하며, 사용자 친화적인 UI/UX를 제공해야 한다
10. 각 섹션은 bootstrap, flexbox 등을 활용하여 반응형 레이아웃으로 구현되며 AI 기술테크포럼의 브랜드 아이덴티티와 첨부된 배경이미지를 반영한 디자인이 적용된다
10-1. /asset/design 내 제공되는 KB 컬러가이드와 최종시안, y배너 이미지파일을 반영한다.
10-2. /asset/fonts 에서 제공하는 웹폰트를 사용한다.
11. event 섹션의 경품 이벤트는 인증된 사용자(참가신청완료)에 한하여 접속가능하며, 사전접수(사용자인증) 후 참여할 수 있다
12. event 세션은 qr코드를 통해 접근되며 qr코드에 추가되는 파라미터에 의해 참여부스를 구분하고 순차적으로 스탬프를 완성하는 방식으로 운영된다
13. 체험부스에 비치하는 QR코드는 관리자페이지에서 부스별 생성버튼을 통해 제공되며 30초 단위로 자동으로 갱신되어야 하며 유효시간을 표시하여 갱신예정을 확인할 수 있어야 한다.
13-1. 해당QR에 포함되는 내용은 각 부스별 고유코드를 현재시간, 부스코드 를 포함한 일련의 스트링을 backend에서 암호화하고 이를 파라미터화 하여 제공하고 frontend에서 인위적으로 변조할 수 없도록 하여 api에서 복호화 하도록한다.
14. 체험부스는 총 4개로 구성되며 3개의 삼각형형태로 구성된 스탬프 아이콘을 비활성상태로 배치하고, 4개중 순서에 상관없이 3개의 각 부스 방문 시 관리자페이지를 통해 생성,비치된 qr코드를 스캔하여 스캔된 url페이지에 접속시 qr코드내 포함된 파라미터를 인증정보와 함께 api에 호출한다.
14-1. 해당 api에서는 복호화된 호출parameter를 바탕으로 해당 부스의 참여를 확정하고 순차적으로 아이콘을 활성화하는 방식으로 구현된다
14-2. backend에서 인증과정에서 복호화된 파라미터에 포함된 현재시간보다 1분이상 경과되면 새로운 qr을 스캔하도록 안내하여야 한다.
14-3. 체험부스 3개이상 참여하여 스탬프아이콘이 모두 활성화된 개인에게는 경품교환 자격이 부여된다.
15. 3개의 아이콘이 활성화되어 경품수령 자격이 부여된 사용자는 고유QR을 생성하는 버튼을 표시한다.
 - QR은 암호회된 인증정보, 참여한 부스정보, 현재시간을 포함하여 실시간으로 생성한다
15-1. 경품수령은 관리자의 모바일기기에서 사용자가 생성한 고유QR을 인식하고 인식된 QR코드 값을 api에 전송함으로써 인증정보를 확인하고, 경품지급사실을 기록하고 경품지급 할 수 있도록 표시한다.
15-2. 사용자는 경품지급용 qr생성시 부터 websocket에 연결하여 관리자의 경품지급 인증절차 데이터를 실시간으로 수신하여 지급처리 완료시 qr코드를 지우고 축하 메세지를 표시한다.
16. 각 세션별 설문참여 qr을 통해 피드백을 수집하는 기능이 포함된다.
17. 설문참여 qr코드는 각 세션 종료 후 프리젠테이션 화면에 노출할 수 있도록 사전에 생성할 수 있다.
18. 설문참여는 익명으로 진행되며, 별도의 인증 절차 없이 참여할 수 있다.
19. 설문참여 qr코드는 각 세션별로 고유한 링크를 포함하며, 링크는 설문조사 페이지로 이동한다. - 별도의 유효기간이 없으며 관리자페이지에서 세션별 QR코드를 생성할 수 있다
20. 설문조사 페이지는 모바일에 최적화 하고 각 세션의 정보를 표시하고 다음 문항에 대해 10점척도로 설문할 수 있도록 구성한다.
 - 문항별 점수선택은 수치로 1---5---10 으로 표시하고 슬라이드 또는 라디오 모두 선택가능하다
 - 문항정보:
  선택형(필수입력) - 강의만족도, 강사만족도, 현업적용도
  서술형(선택입력) - 만족한점, 아쉬운점
21. 관리자 페이지는 반응형 웹으로 구현되며, 모바일웹 환경에서도 원활히 이용할 수 있도록 하고, 별도의 인증 절차를 거쳐 접근할 수 있으며, dashboard를 통해 설문결과, 부스 참여현황, 전체 참가현황을 시각화 하여 표시한다.
 - 관리자 페이지의 주요 기능은 설문조사 결과 조회, 데이터 시각화, 체험부스 참여현황, 인증된 사용자 현황이다.
 - 설문결과는 dashboard 상 전체참여인원수를 표시하고, 세션별 강의만족도, 강사만족도 평균점을 도넛그래프로, 현업적용도를 세션별 바 그래프로 표시한다.
 - 서술형 설문은 [전체보기]를 통해 리스트업 된 목록을 조회할 수 있다.
 - 참가신청현황, 부스별 참여현황, 경품지급현황은 각 부스의 참여인원을 시각화하여 표시한다
 - 각각의 상세 참여목록은 [전체보기]를 통해 조회하며 참가신청현황,부스참여현황, 경품지급현황을 실명을 기반으로 부서명,호칭,직원명을 기본표시한다.
 - 관리자 id/pw - foruma, forumPassPass

22. 관리자 화면에서는 참가신청(사전접수)자 전체를 대상으로 경품 추첨기능을 제공한다.
 - 참가신청자에게는 순차적으로 고유번호를 부여하고 부여하고 이벤트 페이지를 통해 본인의 경품추첨번호를 확인할 수 있다
 - 경품추첨은 횟수에 상관없이 번호 추첨기능을 제공한다
 - 번호추첨기능은 전체 참가자 (max 추첨번호) 를 기반으로 자릿수를 구성하고 해당 자릿수별 회전판을 생성하여 각 자릿수별 0~9에 해당하는 회전판을 돌려 추출하는 방식으로 최종 추첨번호를 제시한다.
