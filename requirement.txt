스마트폰 모바일웹을 이용하여 아래와 같은 조건의 기능을 수행하는 웹어플리케이션을 만들고자 한다.
[개발환경]
frontend : bootstrap, css를 이용하는 html 기반 모바일 웹어플리케이션
backend : nodejs , sqllite 이용하여 사용자 300명이내의 소규모 관리기능을 포함하는 was 

[인증 기능 요구사항]
1. 사용자는 홈페이지의 특정 path 이하 페이지에 접근하는 경우 id, pw를 입력하여 최초 1회 인증절차를 거친다. 이후 로컬스토리지에 저장된 인증정보를 통해 api를 호출한다. 홈페이지 접속시 스토리지의 인증key 존재 여부에 따라 인증여부를 확인한다. 해당 키가 없으면 인증절차로 이동한다.
 특정path 예시: /app/event/
2. 인증정보는 JWT 토큰 형식으로 발급되며, 토큰 만료시간은 1개월이다.
3. 인증이 완료된 사용자는 특정 api를 호출하여 데이터를 조회할 수 있다.
4. api 호출 시 로컬스토리지에 저장된 JWT 토큰을 헤더에 포함하여 전송한다.
5. api 호출 결과는 JSON 형식으로 반환되며, 화면에 데이터를 렌더링한다.
6. 토큰이 만료된 경우, 사용자는 다시 인증절차를 거쳐야 한다.
7. 인증절차 및 api 호출 시 네트워크 오류가 발생할 경우, 사용자에게 적절한 오류 메시지를 표시한다.

[Authentication]
    id=직원번호, pw=휴대번호 뒷4자리
    token_expiry=1개월
[API]
    data_endpoint=/api/data
    auth_endpoint=/api/auth
[ErrorMessages]
    network_error=네트워크 오류가 발생했습니다. 다시 시도해주세요.
    token_expired=인증 토큰이 만료되었습니다. 다시 로그인해주세요.
    invalid_credentials=아이디 또는 비밀번호가 올바르지 않습니다.
[Storage]
    token_key=auth_token
    expiry_key=token_expiry_date
[Headers]
    kb-auth=Authorization
    content_type=Content-Type


[업무 요구사항]
1. 1일간 실시되는 AI 기술테크포럼에 대한 랭딩페이지이다
2. 해당 페이지는 행사 안내, 일정, 연사소개, 부스소개, 참가신청, event 등의 섹션으로 구성된다
3. 행사 안내 섹션에는 행사 개요, 일시, 장소 등의 정보가 포함된다
4. 일정 섹션에는 시간대별 세션 정보가 포함되며 youtube 영상 링크가 첨부된다.
5. 연사소개 섹션에는 연사의 사진, 이름, 소속, 소개글이 포함된다
6. 부스소개에서는 해당 부스에서 제공하는 서비스 및 체험행사에 대한 설명이 포함된다
7. 참가신청 섹션에는 사전접수 폼이 포함되어 상위 인증 절차를 진행할 수 있다
8. event 섹션에는 경품 이벤트 안내 및 참여 방법이 포함된다
9. 각 섹션은 모바일웹 환경에 최적화되어야 하며, 사용자 친화적인 UI/UX를 제공해야 한다
10. 각 섹션은 bootstrap, flexbox 등을 활용하여 반응형 레이아웃으로 구현되며 AI 기술테크포럼의 브랜드 아이덴티티를 반영한 디자인이 적용된다
11. 참가신청 폼에는 직원번호와 휴대번호 뒷4자리를 입력받는 필드가 포함된다
12. event 섹션의 경품 이벤트는 참가자에 한하며, 사전접수 후 참여할 수 있다
13. event 세션은 qr코드를 통해 접근되며 qr코드에 추가되는 파라미터에 의해 참여부스를 구분하고 순차적으로 스탬프를 완성하는 방식으로 운영된다
14. 체험부스는 총 5개로 구성되며 3개의 아이콘을 비활성상태로 배치하고, 5개중 순서에 상관없이 3개의 각 부스 방문 시 qr코드를 스캔하여 해당 아이콘을 활성화하는 방식으로 구현된다
15. 3개의 아이콘이 활성화되면 경품 이벤트 참여가 가능한 추첨 번호를 부여한다.
16. 경품 이벤트 참여 방법 및 추첨 번호는 event 섹션에 안내된다.
17. 각 세션별 설문참여 qr을 통해 피드백을 수집하는 기능이 포함된다.
18. 설문참여 qr코드는 각 세션 종료 후 프리젠테이션 화면에 노출된다.
19. 설문참여 도 하나의 체험부스 로 간주되며, 설문참여 qr코드 스캔 시 해당 아이콘이 활성화된다.
20. 설문참여는 익명으로 진행되며, 별도의 인증 절차 없이 참여할 수 있다.
21. 설문참여 qr코드는 각 세션별로 고유한 링크를 포함하며, 링크는 설문조사 플랫폼으로 연결된다.
22. 설문조사 플랫폼은 자체 구축하여 각 세션별 강의만족도, 강사만족도, 개선사항을 수집하도록 문항을 구성하고, 결과는 관리자 페이지에서 확인할 수 있도록 한다.
23. 관리자 페이지는 별도의 인증 절차를 거쳐 접근할 수 있으며, 설문조사 결과를 시각화하여 제공한다.
24. 관리자 페이지는 반응형 웹으로 구현되며, 모바일웹 환경에서도 원활히 이용할 수 있도록 한다.
25. 관리자 페이지의 주요 기능은 설문조사 결과 조회, 데이터 시각화, 체험부스 참여현황, 인증된 사용자 현황이다.
26. 사용자인증 후 행사장 입장 확인은 QR코드 스캔을 통한 url 접근이며, 해당 url에 입장 파라미터를 포함하는 qr을 스캔함으로써 확인 한다.
27. event 영역에서 각 개인은 입장qr 확인여부, 각 부스참여 여부, 설문참여 여부를 표시하고 부스참여와 설문참여를 종합하여 3개의 아이콘 활성화 상태를 관리한다.
28. event 영역의 입장확인과 3개 아이콘이 모두 활성화된 개인에게는 경품 이벤트 참여 자격이 부여된다.
29. 관리자 페이지에서 각 개인의 입장확인 현황, 부스참여 현황, 설문참여 현황을 조회할 수 있다.
30. 관리자 페이지의 데이터는 api를 통해 제공되며, 인증된 관리자만 접근할 수 있다.

[EventPage]
    sections=행사 안내, 일정, 연사소개, 부스소개, 참가신청, event
[EventDetails]
    overview=행사 개요, 일시, 장소
    schedule=시간대별 세션 정보, youtube 영상 링크
    speakers=사진, 이름, 소속, 소개글
    booths=서비스 설명, 체험행사
    registration=사전접수 폼
    event_info=경품 이벤트 안내, 참여 방법
[UI/UX]
    mobile_optimized=true
    user_friendly=true
[Paths] 
    landing_page=/app/intro/
    auth_page=/app/event/auth
    data_page=/app/event/data   
[YouTube]
    video_link_format=https://www.youtube.com/watch?v=  
[FormFields]
    registration_name=직원번호
    registration_phone=휴대번호 뒷4자리
[EventRules]
    prize_eligibility=참가자에 한함
    participation_method=사전접수 후 이벤트 섹션 참조
